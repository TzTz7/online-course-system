快速实现（在线课堂）建议 — 最小可行方案与进阶
1) 最小可行（简单快速）
   - 上传：前端 HTML 表单 + Flask 接收存到磁盘或对象存储（S3/OSS）。  
   - 播放：使用 HTML5 <video> 直接播放 mp4（浏览器支持的编码）。  
   - 优点：实现快，部署简单；缺点：无法自适应码率，带宽不友好。

2) 推荐做法（生产/可扩展）
   - 存储：用户上传到后端 → 后端把原始文件上传到对象存储（S3/GCS/Azure Blob）。  
   - 异步转码：后端把转码任务丢到队列（Celery / RQ / Kubernetes job），用 FFmpeg 转成 HLS（分片 + m3u8）并生成多个码率。参考 python-flask-ffmpeg、BetterTGStreamer。  
   - 分发：把 HLS 文件存回对象存储并配合 CDN（CloudFront 等）加速。或直接通过 presigned URL 供播放器访问（参考 flask-s3-video-player）。  
   - 前端播放器：用 Plyr、video.js 或 hls.js 播放 HLS（兼容大部分浏览器和移动端）。  
   - 安全与控制：课程权限控制（签名 URL / token 验证），播放统计（用于进度/记录观看情况），DRM 可选。  
   - 可选：实现断点续传（tus 或前端分片上传）、生成缩略图、字幕/转录（Whisper）、视频水印/防盗链。

实现要点速查（对应上面仓库）
- 上传接收与保存：参照 Video-Sphere、VaultTube、flask-server（简单上传示例）。  
- S3 presigned 播放：参照 flask-s3-video-player。  
- 转码与 HLS 生成：参照 python-flask-ffmpeg、BetterTGStreamer。  
- 异步处理/队列：参照 microservice-app（Celery 示例）。  
- 前端播放：用 hls.js + video.js，配合生成的 m3u8。
